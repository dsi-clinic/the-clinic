# AGENTS.md

## Repository Overview

This repository hosts a **Jekyll-based GitHub Pages website** for the University of Chicago Data Science Clinic at `https://clinic.ds.uchicago.edu`.

## How the Site Runs

### Tech Stack
- **Static site generator**: Jekyll with the `minimal` remote theme
- **Hosting**: GitHub Pages (auto-deployed on push to `main` via `.github/workflows/jekyll.yml`)
- **Local development**: Docker container (Ruby 3.1 + Python via `uv`)
- **Data pipeline**: Python scripts generate markdown from YAML project data

### Local Development

Always use Docker via the Makefile — do not use the local environment directly:

```bash
make          # Build Docker image and serve site on port 4000
make validate # Validate project YAML data with Pydantic
make projects # Regenerate projects/index.md from YAML data
make clean    # Clean Docker resources
make rebuild  # Full clean rebuild and serve
make sitemap  # Generate site map via Graphviz
```

### Build Flow

1. Docker builds image with Ruby/Jekyll + Python (`uv`) environment
2. `bundle install` installs gems; `uv sync` installs Python deps
3. Jekyll serves with `--livereload` on port 4000
4. On push to `main`, GitHub Actions builds and deploys to Pages

### Key Configuration Files
- `_config.yml` — Jekyll config (theme, plugins, site URL)
- `Dockerfile` — Container definition
- `Makefile` — Build automation
- `Gemfile` — Ruby dependencies (`github-pages` gem)
- `pyproject.toml` — Python dependencies (PyYAML, Pydantic, etc.)

## Conventions

- **YAML naming**: Quarter files use `season_year.yaml` format (e.g., `spring_2025.yaml`)
- **One-pager PDFs**: Stored in `projects/one-pagers/{year}-{season}/`
- **Rubrics**: All in `rubrics/` as markdown files
- **Templates**: All in `templates/` as markdown and/or `.docx` files
- **Do not edit `projects/index.md` directly** — it is generated by `generate_index_md.py`; edit the YAML source data instead
